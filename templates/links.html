Homepage (practice)

This is an anchor. It is defined by using the <a> tag. By itself it doesn't do anything. Go, try it.

To send you anywhere using an anchor I need to use an attribute within the tag I'm defining. This attribute is the href attribute (from hypertext-reference, most probably).

I can add any reference I want within the anchor. We will chose a reference to the site used to learn these skills. That'd be The Odin Project. This hypertext referencing, is important to say, is what makes the web, the web. That is a web of links that reference each other and we can access.

Previous link I used, sorry, anchor is somewhat annoying to click. If you do, it will open the webpage on the same tab/windows you're using to read this. To avoid doing that, the best option is to configure another attribute of the anchor tag: the target.

That, would make this link here to open in a new tab. We added another attribute in the previous tag: rel. We set it up to the string "noopener noreferrer". noopener will stop any javascript execution that the visiting page could exert on the referring page. noreferrer will include this behavior plus privacy. By using noreferrer the visited page will not know where we came from when clicking into the link.

Absolute and Relative Links

Absolute links have an scheme followed by a domain. This will usually be in the format of: scheme://domain/path/inside/the/domain.

Relative links will only have a relative path. For example, if I use the value "text" into this link, it will take you to the page where I practiced text and text formatting in HTML.

Pages shouldn't be thrown all in the same folders. They should be, hopefully, ordered categorically. Imagine I want to access a page at "https://garciahahn.com/self-learning-webdev/pages/about", for that, I only need to add "pages/about" to my href attribute. In The Odin Project, they do this the old way by serving files directly.
