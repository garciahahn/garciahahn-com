Images (Woohoo!)

Images use a void element. The void element in question is: <img>. By assigning an image in the src attribute, either using a relative or an absolute link, we can straight up render an image.

Accessing the parent directory should be as easy as prepending a double dot '..' to the file search. Nevertheless, given how Flask manages static resources, I don't know if that would be something useful for this specific use case

It is important to add the alt attribute in the images. This is important for two reasons:

  1. If the resource (the image) is not available, you'll be able to at least show that alt text.
  2. If someone that is vision impaired is using a screen-reader, the image will be represented by the string that you store in the alt attribute.
